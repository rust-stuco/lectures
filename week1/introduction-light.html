Why are you all here?

This section is heavily influenced by Jon Gjengset's talk about Considering Rust: https://www.youtube.com/watch?v=DnT-LUQgc7s

There is a saying: Fast, reliable, or productive - pick three!

* "is blazingly fast and memory-efficient" (**_fast_**) * "enabl[es] you to eliminate many classes of bugs at compile-time" (**_reliable_**) * "has great documentation, a friendly compiler with useful error messages, and top-notch tooling" (**_productive_**) * "[is] A language empowering everyone to build reliable and efficient software"

Built-in unit testing and documentation, friendly errors

The borrow checker can feel "unfair" and hard Requires thinking a lot up front before you write down code

Rust is not a magic bullet to solve everyone's problems!

Speaker note: "built-in" => included with all Rust distributions, eliminates need for third-party build systems

* To build for release, run `cargo build --release`

* Note that if you compiled with `cargo build` right before, you wouldn't see the `Compiling hello_cargo ...` line

* To run in release, run `cargo run --release`
