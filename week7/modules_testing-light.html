In terms of privacy, unit tests are treated just as any other function. Excerpt from the Rust Book: There's debate within the testing community about whether or not private functions should be tested directly, and other languages make it difficult or impossible to test private functions. If you don't think private functions should be tested, there's nothing in Rust that will compel you to do so.

By "any other code" we mean code that was written by some other developer using your library crate.

You can treat the `tests` directory almost exactly the same as the `src` directory, and you can also use the alternate module file naming that we talked about earlier in the lecture.
